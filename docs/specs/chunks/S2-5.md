## Mission

Reorganize the 3 existing instrument detail routes (`/stocks/[symbol]`, `/etf/[ticker]`, `/funds/[id]`) into the locked 6-tab structure. Reuse the existing hero blocks (`StockHeroBlock`, `FundHeroBlock`) but extend them to carry the shared 4-lens card + composite action + conviction score + weight band.

## Scope

1. **Sticky hero block (shared shape, per-asset fields):**
   - Symbol / name / sector / current price + change
   - 4-lens card with benchmark switcher (live values + per-lens signal flags from `/api/v1/lens/{scope}/{id}`)
   - Composite action badge (BUY / HOLD / WATCH / AVOID / SELL) + one-line reason
   - Conviction score 0-100 + suggested weight band

2. **Six sub-tabs** (names identical across all 3 routes; fields per §4.6 table):
   - Overview
   - Selection
   - Weighting
   - Timing
   - Events
   - Backtest

3. **Backtest tab** reads `atlas_signal_history` precomputed (§8 Q10). If gaps → `insufficient_data` envelope → `<EmptyState>`. Do NOT run a live backtest.

4. **Events tab** surfaces insider trades + corporate actions + earnings (stocks); holdings rebalance (ETFs); manager changes + scheme merges + exit-load (MFs). All from existing JIP tables — no new fields.

5. **Reuse** `frontend/src/components/lenses/FourLens.tsx` — extend props for per-lens signal flags, do not fork.
