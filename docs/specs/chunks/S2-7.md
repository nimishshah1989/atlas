# S2-7 — View 4: Book + Action Queue (`/book`)

**Slice:** `docs/specs/slices/s2-drill-spine.md` (read §4.5)
**Status:** PENDING  •  **Model:** sonnet  •  **Estimate:** $6 / 6h
**Depends on:** S2-1, S2-6

## Mission

Build the Book view at `/book`. Collapses 8 MarketPulse pages (Recommendations + Microbaskets + Portfolios + Actionables + Actioned + Approved + Trade + Performance) into one page with Holdings / Watchlist tabs and a 3-section stack.

## Scope

1. **Route:** `frontend/src/app/book/page.tsx`.

2. **Tab bar:** `Holdings | Watchlist`. Same component shell, different data source (both from `/api/v1/book` built in S2-0).

3. **Section 1 — Action Queue** (top):
   - EXIT (red), WARN (amber), CONFIRM (green) signals on current positions.
   - Sort: `signal_strength × position_size` descending.
   - **Insider sells** + **corporate actions** overlaid as event flags on the relevant rows (reads from existing JIP tables per §6.2, no new fields).
   - Each row has "Size it" button → opens `<ConvictionDrawer>` from S2-6.

4. **Section 2 — Holdings table:** full 11-column schema (slice §5) + 2 extras specific to Book: `Entry Conviction`, `P&L`. Document the extended schema in the component.

5. **Section 3 — Performance attribution:** breakdown by sector / signal type / hold period. Uses atlas's 5y history. If gaps → `<EmptyState>` per two-idiom rule.

6. **Watchlist tab** — same 3-section shell but "Action queue" becomes "Near-entry signals" (WARN + ENTRY) and "Performance attribution" becomes "What if I had bought on signal N weeks ago" (reads `atlas_signal_history`, same read-only history as S2-5 backtest).
