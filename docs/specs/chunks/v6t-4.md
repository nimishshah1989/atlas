## Goal

Surface TradingView TA across the two list views that already exist:

1. **Market tab** (`<StockTable />`) — new "TV" column showing a compact 1D
   recommendation chip per row. Uses cache-only bulk lookup; no per-row bridge
   calls. Uncached symbols render "—".
2. **Watchlist page** (`/pro/watchlists`) — replace the now-dead "Sync to TV"
   button with **"Refresh TV signals"**. Click → fires `/api/tv/ta/{symbol}`
   in parallel for each symbol in the watchlist, then renders a mini chip next
   to each symbol chip in-card. The existing `POST /{id}/sync-tv` route was
   removed in V6T-2 and must also be removed from the frontend API client.

All colour + label logic reuses `classifyTvScore` from V6T-3's
`frontend/src/lib/tv.ts` — no duplicated thresholds.

## Background

**What exists:**

- `frontend/src/lib/tv.ts::classifyTvScore(raw)` → `{label, score, className, display}` (V6T-3).
- `frontend/src/components/deepdive/TVConvictionPanel.tsx` consuming it (V6T-3).
- `GET /api/tv/ta/{symbol}` returns per-symbol TA (read-through cache + bridge on miss). Fine for watchlist (≤20 symbols, user-initiated). Not fine for market tab (≤500 symbols — would hammer TradingView).
- `atlas_tv_cache` table — PK `(symbol, data_type, interval)`, `tv_data JSONB`, `fetched_at`.
- `POST /api/watchlists/{id}/sync-tv` → 404 (V6T-2 removal).
- `syncWatchlistToTv()` in `frontend/src/lib/api-watchlists.ts` still calls the dead route.

**Shape of `tv_data` for `data_type='ta_summary'`:** the dict returned by
`tradingview_screener` (V6T-2), keys include `Recommend.All`, `Recommend.MA`,
`Recommend.Other`, `RSI`, `close`, `volume`, etc. `Recommend.All` is the 1D
composite score used for all chips in this chunk.

**Why a new bulk endpoint:** firing 500 per-symbol `/api/tv/ta/{symbol}` calls
from the market tab would slam TradingView (bridge call per cache miss). Market
tab chips are an ambient signal, not a trigger to fetch — so we add a pure
cache-lookup endpoint that returns whatever is already in `atlas_tv_cache`,
never calls the bridge, never triggers a background refresh.

## Files to Change

### 1. `backend/routes/tv.py` — Add bulk cache-only route

```python
@router.get("/ta/bulk")
async def get_ta_bulk_cache(
    symbols: str,  # comma-separated, e.g. "RELIANCE,TCS,INFY"
    exchange: str = "NSE",
    interval: str = "1D",
    db: AsyncSession = Depends(get_db),
) -> dict[str, Any]:
    """Pure-cache lookup for 1D TA across many symbols.

    Returns a §20.4 envelope with an `items` array — one entry per requested
    symbol. Uncached symbols yield `{symbol, tv_ta: null, fetched_at: null}`.
    Does NOT call the bridge and does NOT trigger background refresh — use
    `GET /api/tv/ta/{symbol}` for single-symbol reads-through behaviour.

    Query params
    ------------
    symbols  : comma-separated symbols. Case-insensitive; upper-cased. Hard cap
               of 500 to keep the WHERE clause sane; 400 otherwise.
    exchange : currently informational (cache PK is (symbol, data_type, interval)).
    interval : '1D' by default; passed through to the cache PK.
    """
    from backend.db.tv_models import AtlasTvCache
    from sqlalchemy import select

    raw = [s.strip().upper() for s in symbols.split(",") if s.strip()]
    if not raw:
        raise HTTPException(status_code=400, detail="symbols required")
    if len(raw) > 500:
        raise HTTPException(status_code=400, detail="max 500 symbols per call")
    # Dedupe while preserving order so the response array mirrors the request.
    seen: set[str] = set()
    requested: list[str] = []
    for s in raw:
        if s not in seen:
            seen.add(s)
            requested.append(s)

    stmt = select(AtlasTvCache).where(
        AtlasTvCache.symbol.in_(requested),
        AtlasTvCache.data_type == "ta_summary",
        AtlasTvCache.interval == interval,
    )
    rows = (await db.execute(stmt)).scalars().all()
    by_symbol = {r.symbol: r for r in rows}

    items: list[dict[str, Any]] = []
    latest: Optional[datetime] = None
    for sym in requested:
        row = by_symbol.get(sym)
        if row is None:
            items.append({"symbol": sym, "tv_ta": None, "fetched_at": None})
            continue
        if latest is None or row.fetched_at > latest:
            latest = row.fetched_at
        items.append(
            {
                "symbol": sym,
                "tv_ta": row.tv_data,
                "fetched_at": row.fetched_at.isoformat(),
            }
        )

    return {
        "data": {"items": items},
        "_meta": {
            "data_as_of": latest.isoformat() if latest else None,
            "data_layer": "near_realtime",
            "cached_count": sum(1 for it in items if it["tv_ta"] is not None),
            "requested_count": len(requested),
        },
    }
```

Route order note: this sits alongside the existing `/ta/{symbol}` route. FastAPI
matches literal `/ta/bulk` before the `{symbol}` path parameter — verify in the
test (symbol literally named "bulk" would be a cache miss, never a route
conflict, because `bulk` is declared first; if order matters at runtime, the
implementer must register `/ta/bulk` BEFORE `/ta/{symbol}`).

### 3. `frontend/src/lib/api.ts` — Add bulk-cache fetcher

Append below the existing deep-dive exports:

```ts
export interface TvBulkItem {
  symbol: string;
  tv_ta: Record<string, number | string | null> | null;
  fetched_at: string | null;
}

export interface TvBulkResponse {
  data: { items: TvBulkItem[] };
  _meta: {
    data_as_of: string | null;
    data_layer: string;
    cached_count: number;
    requested_count: number;
  };
}

export async function getTvBulkCache(
  symbols: string[],
  interval = "1D"
): Promise<TvBulkResponse> {
  if (symbols.length === 0) {
    return {
      data: { items: [] },
      _meta: { data_as_of: null, data_layer: "near_realtime", cached_count: 0, requested_count: 0 },
    };
  }
  const qs = new URLSearchParams({
    symbols: symbols.join(","),
    interval,
  });
  const res = await fetch(`${API_BASE}/api/v1/tv/ta/bulk?${qs.toString()}`);
  if (!res.ok) throw new Error(`bulk tv cache: ${res.status}`);
  return res.json();
}
```

Use the same `API_BASE` + fetch pattern as existing `api.ts` exports.

Add `getTvTa(symbol)` to `frontend/src/lib/api.ts`:

```ts
export interface TvTaResponse {
  data: {
    symbol: string;
    exchange: string;
    interval: string;
    recommendation_1d: string | null;
    tv_ta?: Record<string, number | string | null> | null;
    [k: string]: unknown;
  };
  _meta: { data_as_of: string; is_stale: boolean; data_layer: string };
}

export async function getTvTa(symbol: string, interval = "1D"): Promise<TvTaResponse> {
  const res = await fetch(`${API_BASE}/api/v1/tv/ta/${encodeURIComponent(symbol)}?interval=${interval}`);
  if (!res.ok) throw new Error(`tv ta ${symbol}: ${res.status}`);
  return res.json();
}
```

Note on the existing TA route shape: `GET /api/tv/ta/{symbol}` returns
`_extract_ta_fields` output which does NOT include the raw `tv_ta` dict by
default — it only exposes pre-processed `recommendation_1d` / `oscillator_score`
/ `ma_score`. The implementer must **either**:

- (a) have the route also include the raw `tv_ta` field alongside the
  extracted ones (additive change to `/api/tv/ta/{symbol}` response), **or**
- (b) have the frontend parse the `recommendation_1d` string ("STRONG_BUY"
  etc.) back into a score via a reverse map.

**(a) is preferred** — additive `tv_ta: tv_data_raw` on the response. It's a
strict superset, no existing callers break, and the chip helper keeps its
single source of truth on the numeric `Recommend.All` field.

## Important Notes

- **No bridge calls from the market tab.** The bulk route reads `atlas_tv_cache` only. Deep-dive visits warm that cache over time. A stock never visited yet shows `—` — that's by design.
- **Route ordering matters.** Register `/ta/bulk` BEFORE `/ta/{symbol}` in `backend/routes/tv.py` so FastAPI literal-matches first. Test with `GET /api/v1/tv/ta/bulk?symbols=X` — if it hits the `{symbol}` handler with `symbol="bulk"`, the order is wrong.
- **Additive change to `/api/tv/ta/{symbol}`.** Adding `tv_ta: tv_data_raw` to the existing response is strict-superset and does not break `tv_partial` logic in `stocks.py` (that reads from `tv_cache.get_or_fetch` directly, never from this route). Update `tests/routes/test_tv_routes.py` to assert the new field if it didn't already.
- **`classifyTvScore` is load-bearing.** Do not re-implement thresholds in `TvChip` or in the watchlist card. If thresholds ever change, they change in `lib/tv.ts` once.
- **No new npm deps.** Tailwind + React 19 only.
- **Do not touch `ConvictionPillars.tsx`** — V6T-3 kept the 3-column pillar grid intact deliberately.
- The TV column is the 15th column on a desktop-dense table — the existing `overflow-x-auto` container handles horizontal scroll on narrower widths; no responsive work needed in this chunk.
- Per frontend/AGENTS.md, Next 16.2.3 + React 19.2.4 may differ from training data. This chunk uses only client components, standard hooks, and `URLSearchParams` — no new framework APIs.
