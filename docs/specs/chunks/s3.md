## Goal

Write `docs/specs/v1-criteria.yaml` — the machine-readable form of the 15 V1 completion criteria from `ATLAS-DEFINITIVE-SPEC.md` §24.3. Each criterion becomes one executable check inside the `product` dim. Once this file exists, the product dim returns real numbers (still `gating: false` until V1.6 R1 flips it) and the forge dashboard starts showing V1 completion progress as a 0–100% score the user can watch climb as V1.6 chunks ship.

## Why

Criterion 7 ("≥5 decisions per pipeline run") and criterion 12 ("≥10 intelligence findings stored") are the two that prove V1 is actually intelligent, not just a read-API over JIP. They currently fail silently because nothing generates decisions or findings. Wiring them as executable product-dim checks means the next time anyone claims V1 is done, the scoring engine can contradict them with a file:line.

## Criterion file format

```yaml
version: "1"
slice: "V1"
source: "ATLAS-DEFINITIVE-SPEC.md §24.3"
criteria:
  - id: v1-01
    title: "stocks/universe returns valid data matching contract"
    severity: critical
    check:
      type: http_contract
      url: "http://127.0.0.1:8010/api/v1/stocks/universe"
      contract: "backend.models.stocks.StockUniverseResponse"
      max_latency_ms: 2000
  - id: v1-07
    title: "≥5 decisions per pipeline run"
    severity: critical
    check:
      type: sql_count
      query: "SELECT COUNT(*) FROM atlas_decisions WHERE created_at >= now() - interval '1 day'"
      min: 5
  - id: v1-12
    title: "≥10 findings stored after first pipeline run"
    severity: critical
    check:
      type: sql_count
      query: "SELECT COUNT(*) FROM atlas_intelligence WHERE created_at >= now() - interval '1 day'"
      min: 10
  # ...13 more
```

Supported check types for V1:
- `http_contract` — hit URL, validate response against Pydantic model, assert latency.
- `sql_count` — run query, assert row count min/max/equal.
- `sql_invariant` — run query, assert single scalar value against rule (e.g., `sector_sum_stock_count BETWEEN 2600 AND 2800`).
- `python_callable` — dotted path to a function that returns `(bool, str_evidence)`. Escape hatch for anything the declarative types can't express (e.g., "no float in any financial calculation" — that's an AST check).
- `file_exists` — path, optional min_size_bytes.
