## Objective

Install SWR and create the shared infrastructure every S1 page chunk depends on:
a typed fetch wrapper (`apiFetch`), a SWR hook (`useAtlasData`) that mirrors the
four-state contract of `atlas-data.js` §6, a set of UI primitives mirroring the
four `atlas-states.js` renderers, and augmented formatting utilities. No page code
in this chunk — only shared lib/hooks/components.

## Domain constraints

- No synthetic/hardcoded data in any component — all values come from props or hook.
- Financial values displayed via `formatCurrency` / `formatPercent` / `formatIndianNumber` — never raw numbers with rupee sign hardcoded.
- `AtlasMeta.data_as_of` must be displayed as DD-MMM-YYYY IST, never raw ISO string.
- `STALENESS_THRESHOLDS` values must exactly match atlas-states.js values (intraday=3600, eod_breadth=21600, daily_regime=86400, fundamentals=604800, events=604800, holdings=604800, system=21600).
- Do NOT modify any existing file outside this chunk's listed new files — the existing `api.ts`, `format.ts` may only be augmented with new exports, not have existing exports modified.
