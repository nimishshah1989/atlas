# Chunk S1 — Scoring engine rewrite (new 7 dims, no composite, per-dim 80% gate)

**Depends on:** FD-3 (latest forge dashboard V2 chunk)
**Blocks:** S2, S3, S4, all V1.6 product chunks
**Complexity:** L (4–6h)
**Dimension:** code, architecture, devops

---

## Goal

Rewrite `.quality/checks.py` so it emits **seven independent per-dimension scores with no composite**, enforces a **per-dimension 80% floor**, and uses the **passed/eligible** formula the user specified. Replace the legacy dim set (`security, code, architecture, api, frontend, devops, docs`) with the new target set (`security, code, architecture, api, frontend, backend, product`). Fold high-value DevOps + Docs checks into `backend` and `architecture`; drop the ceremony ones. Add two new dimensions: `backend` (data correctness, business logic, DB health, pipeline health) and `product` (executable V1 completion criteria sourced from `docs/specs/v1-criteria.yaml` — file is stubbed in S1, populated in S3).

## Why

The current engine emits a weighted composite "overall 90/100" that hid the fact that V1 was shipped with zero agents, zero decisions, and zero intelligence findings. A composite is not a gate — it's an average that lets strong dims carry weak ones. The user's rule: every dim on its own, 80% floor, no averaging.

## Files

### New
- `.quality/dimensions/__init__.py` — dimension registry + shared types (`DimensionResult`, `CheckResult`).
- `.quality/dimensions/product.py` — loads `docs/specs/v1-criteria.yaml` (tolerates missing file in S1), each criterion → one check, `passed/eligible × 100`.
- `.quality/dimensions/backend.py` — data correctness (row-count invariants, sector-sum-equals-universe), business logic (RS momentum formula hand-check, pipeline idempotence probe), DB health (alembic-check, FK-index presence, N+1 probe), pipeline health (daily pipeline run completes, atlas_intelligence + atlas_decisions have rows after run).

### Modified
- `.quality/checks.py` — gut the composite. Emit `DimensionResult` per dim with `gating: bool`. Remove `dim_devops()`, `dim_docs()`. Keep `dim_security()`, `dim_code()`, `dim_architecture()`, `dim_api()`, `dim_frontend()`. Import and call `dim_backend()`, `dim_product()`. Fold high-value DevOps checks (`docker_build_ok`, `alembic_apply_clean`, `pip_audit`) into `backend`. Fold high-value Docs checks (`readme_present`, `claude_md_present_and_live`, `adr_count`) into `architecture`. Drop the rest (SSL expiry, bundle size ceremony, CI workflow ping — these belong in infra monitoring, not per-chunk gates).
- `.quality/report.json` — new shape: `{"dims": {"security": {"score": N, "gating": true, "passed": N, "eligible": N, "checks": [...]}, ...}, "generated_at": "..."}`. No top-level `overall`.
- `orchestrator/runner.py` — the `_run_quality_gate()` method must read the new report shape, iterate dims, and fail the chunk if **any gating dim < 80**. Non-gating dims are logged but do not block.
- `orchestrator/plan.yaml` — replace `settings.quality.min_overall` + `min_dimensions` with `settings.quality.min_per_gating_dim: 80`. Remove `devops` and `docs` floor entries. Add `backend` and `product` as `gating: false` until V1.6 R1 flips them.
- `frontend/src/app/forge/page.tsx` (+ dashboard components) — render **7 independent bars**, remove the "overall" number from every surface. Grey-out non-gating dims with an "informational" badge.

### Deleted
- None. Legacy dim functions are removed from `checks.py` but the file stays.

## Appendix A — New backend dimension checks

1. `b1_alembic_head_matches_models` — `alembic check` passes.
2. `b2_all_fks_indexed` — SQL introspection: every `atlas_*.*_id` FK has an index.
3. `b3_no_float_in_financial_columns` — SQL: no `double precision` in atlas_* tables except deliberately-allowed columns.
4. `b4_row_count_invariants` — sector rollup `SUM(stock_count)` equals universe count ± 2%.
5. `b5_rs_momentum_hand_check` — pick a fixed symbol, compute RS momentum via code and via raw SQL, assert within 0.01.
6. `b6_pipeline_idempotent` — if `backend/pipelines/daily.py` exists: run twice, row counts equal. If file missing: `eligible=0` (informational only).
7. `b7_intelligence_writes_on_run` — if pipeline file exists: after run, `SELECT count(*) FROM atlas_intelligence > 0`. Informational until V1.6.
8. `b8_decisions_generated_on_run` — same shape as b7 for `atlas_decisions`.
9. `b9_no_n_plus_1_in_hot_routes` — hit `/stocks/universe` and `/stocks/sectors` with query counter; each route < 25 queries.
10. `b10_pip_audit_clean` — folded from devops.

b6, b7, b8 return `eligible=0` until the pipeline file exists. This is the mechanism that lets S1 pass while V1.6 is pending.
