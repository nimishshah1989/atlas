## Goal

Close the V6 TradingView slice the same way V2/V3/V4/V5 were closed:
transcribe its completion criteria into a testable YAML, dispatch them
through the existing check-types runtime, wire them into the product
quality dim, and ship a validator script. Then confirm all 7 quality
dims ≥ 80 and the frontend production build succeeds.

This is the **polish chunk** for V6 — it adds no product features and
no routes. It turns "we shipped V6-1..V6-9 + V6T-1..V6T-4" into
"we can prove, in one command, that V6 is complete and stays complete".

---

## Background

V2/V3/V5 all followed the same polish-chunk pattern:

- `docs/specs/vN-criteria.yaml` — YAML catalogue of every shippable
  contract, each with a `check.type` that the existing
  `.quality/dimensions/check_types/` dispatch table knows how to run.
- `.quality/quality_product_checks_vN.py` — stdlib-only callables for
  `python_callable` criteria (endpoint reachability, no-float AST scan,
  secret-present scan, etc.).
- `scripts/validate-vN.py` — thin wrapper that reads the YAML and prints
  PASS/FAIL per criterion, exits non-zero on any failure.
- Product dim wiring in `.quality/dimensions/product.py` —
  `checks.extend(_extra_criteria_checks(V6_CRITERIA_PATH))`.
- `tests/unit/test_vN_criteria.py` — criteria file is valid YAML, every
  criterion has `id`/`title`/`check`, every check type is in the dispatch
  table, every `python_callable` name resolves.

V6 never got that treatment because the slice was mid-build. With
V6T-4 (TV column + watchlist refresh) landed, V6 is feature-complete
and this chunk locks it in.

**ATLAS-DEFINITIVE-SPEC.md §10** is the TradingView section — not in
`MANDATORY_SECTIONS` of `scripts/check-spec-coverage.py`, so no mandatory
coverage gate, but the criteria still back-reference §10 for traceability.

```python
def check_tradingview_screener_pinned() -> tuple[bool, str]:
    req_path = ROOT / "requirements.txt"
    if not req_path.exists():
        return False, f"requirements.txt not found at {req_path}"
    text = req_path.read_text(encoding="utf-8")
    import re
    match = re.search(r"^tradingview-screener==(\d+\.\d+\.\d+)\s*$", text, flags=re.MULTILINE)
    if match is None:
        return False, "tradingview-screener not pinned to exact version in requirements.txt"
    return True, f"tradingview-screener pinned at {match.group(1)}"
```

For `check_tv_webhook_requires_secret`, POST without the
`X-Tradingview-Secret` header and assert 403:

```python
def check_tv_webhook_requires_secret() -> tuple[bool, str]:
    url = f"{BACKEND_BASE}/api/v1/webhooks/tradingview"
    req = urllib.request.Request(url, method="POST", data=b"{}",
                                 headers={"Content-Type": "application/json",
                                          "User-Agent": "atlas-quality/1.0"})
    try:
        with urllib.request.urlopen(req, timeout=10.0) as resp:
            status = resp.status
    except urllib.error.HTTPError as exc:
        status = exc.code
    except Exception as exc:  # noqa: BLE001
        return False, f"POST {url} unreachable: {str(exc)[:80]}"
    if status == 403:
        return True, f"POST {url} → 403 without secret (gate in place)"
    return False, f"POST {url} → {status} (expected 403 without secret)"
```

For `check_bridge_no_httpx`:

```python
def check_bridge_no_httpx() -> tuple[bool, str]:
    path = ROOT / "backend" / "services" / "tv" / "bridge.py"
    if not path.exists():
        return False, f"{path} missing"
    import ast
    tree = ast.parse(path.read_text(encoding="utf-8"))
    for node in ast.walk(tree):
        if isinstance(node, ast.Import):
            for alias in node.names:
                if alias.name.startswith("httpx"):
                    return False, f"bridge.py imports {alias.name}"
        elif isinstance(node, ast.ImportFrom):
            if node.module and node.module.startswith("httpx"):
                return False, f"bridge.py imports from {node.module}"
    return True, "bridge.py has no httpx import (sidecar removal holds)"
```

## Important Notes

- **No new runtime code.** This chunk adds criteria, checks, a validator,
  and tests. It must not touch `backend/routes/`, `backend/services/`,
  or `frontend/src/` beyond the file_exists criteria looking at them.
  If a criterion fails because a file is missing, the fix is to add the
  file in V6 or V6T, not to weaken the criterion.
- **Criteria are traceability, not rubber stamps.** If a criterion can
  only be satisfied by weakening its check (for example accepting 500s
  from an endpoint), the criterion was wrong — fix the endpoint, not the
  YAML. Consult the user before softening any `severity: critical` check.
- **`check_sync_tv_is_404` is the regression guard for V6T-2.** If
  someone ever re-adds the sync-tv route, this check fails and the
  product dim drops. That is intentional and load-bearing.
- **`check_bridge_no_httpx` is the regression guard for V6T-2's sidecar
  removal.** Same rationale.
- **`check_tv_ta_bulk_endpoint` is the regression guard for V6T-4.**
- **V6 spec section is §10**, not a numbered sub-slice. All criteria
  back-reference `§10`. `scripts/check-spec-coverage.py` does NOT
  require §10 (it is not in `MANDATORY_SECTIONS`), but the references
  still power future audits.
- **pre-existing `test_mf_deep_dive_panel.py` DB timeout failures are
  out of scope.** They hit real RDS and time out; do not gate this chunk
  on them. Exclude with `-m 'not integration'` as the punch list does.
- The frontend production build is load-bearing for this chunk because
  V6T-3 and V6T-4 added new components without an explicit `npm run
  build` check at commit time. V6T-5 is where that proof lives.
- No new npm deps, no new Python deps. All work is config/tests/scripts.
