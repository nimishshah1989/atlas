## Goal

Expose the already-cached TradingView TA data to the FM UI. Render a three-row
coloured summary — Overall / MA / Oscillators — inside the stock Deep Dive panel.
When `tv_ta` is null (bridge unavailable or cache miss), render a single
"TV data unavailable" line in the panel slot. No layout shift, no crash.

Backend already builds `ConvictionPillars.pillar_3` (`PillarExternal`) from
`tv_ta_data` in `backend/core/computations.py::build_conviction_pillars`.
Route `GET /api/stocks/{symbol}` already surfaces it. The frontend type and
render path do not yet read it — this chunk closes that gap.

## Background

**Backend contract (already live, do not touch):**

```python
# backend/models/schemas.py
class PillarExternal(BaseModel):
    tv_ta: Optional[dict[str, Any]] = None
    explanation: str = ""

class ConvictionPillars(BaseModel):
    rs: PillarRS
    technical: PillarTechnical
    institutional: PillarInstitutional
    pillar_3: Optional[PillarExternal] = None
```

`tv_ta` shape is the row returned by `tradingview_screener` (V6T-2). Relevant fields:

| Field          | Range          | Meaning                              |
|----------------|----------------|--------------------------------------|
| `Recommend.All`   | -1.0 … +1.0 | Composite recommendation             |
| `Recommend.MA`    | -1.0 … +1.0 | Moving-average-based recommendation  |
| `Recommend.Other` | -1.0 … +1.0 | Oscillator-based recommendation      |

**Label thresholds (locked in V6T-2 spec, reuse exactly):**

| Score range     | Label        | Colour class (Tailwind)                 |
|-----------------|--------------|------------------------------------------|
| `>= 0.5`        | STRONG_BUY   | `bg-emerald-50 text-emerald-700 border-emerald-200` |
| `0.1 … 0.49`    | BUY          | `bg-emerald-50/50 text-emerald-600 border-emerald-100` |
| `-0.09 … 0.09`  | NEUTRAL      | `bg-gray-50 text-gray-600 border-gray-200` |
| `-0.49 … -0.1`  | SELL         | `bg-red-50/50 text-red-600 border-red-100` |
| `<= -0.5`       | STRONG_SELL  | `bg-red-50 text-red-700 border-red-200` |

`null` / missing score → render "—" + neutral gray border, no label chip.

### 2. `frontend/src/lib/tv.ts` — NEW helper

Single source of truth for score → label + colour mapping so the logic is
testable and reusable by V6T-4 (Stock table mini chips).

```ts
// frontend/src/lib/tv.ts
export type TvLabel = "STRONG_BUY" | "BUY" | "NEUTRAL" | "SELL" | "STRONG_SELL";

export interface TvChip {
  label: TvLabel | null;   // null when score is null/undefined/NaN
  score: number | null;
  className: string;       // Tailwind classes for badge + border
  display: string;         // "+0.62" or "—"
}

const CLASS_BY_LABEL: Record<TvLabel, string> = {
  STRONG_BUY: "bg-emerald-50 text-emerald-700 border-emerald-200",
  BUY: "bg-emerald-50/50 text-emerald-600 border-emerald-100",
  NEUTRAL: "bg-gray-50 text-gray-600 border-gray-200",
  SELL: "bg-red-50/50 text-red-600 border-red-100",
  STRONG_SELL: "bg-red-50 text-red-700 border-red-200",
};
const NEUTRAL_CLASS = "bg-white text-gray-400 border-gray-200";

export function classifyTvScore(raw: unknown): TvChip {
  const score = typeof raw === "number" && Number.isFinite(raw) ? raw : null;
  if (score === null) {
    return { label: null, score: null, className: NEUTRAL_CLASS, display: "—" };
  }
  let label: TvLabel;
  if (score >= 0.5) label = "STRONG_BUY";
  else if (score >= 0.1) label = "BUY";
  else if (score > -0.1) label = "NEUTRAL";
  else if (score > -0.5) label = "SELL";
  else label = "STRONG_SELL";
  const sign = score > 0 ? "+" : "";
  return {
    label,
    score,
    className: CLASS_BY_LABEL[label],
    display: `${sign}${score.toFixed(2)}`,
  };
}
```

### 3. `frontend/src/components/deepdive/TVConvictionPanel.tsx` — NEW

```tsx
"use client";

import { classifyTvScore } from "@/lib/tv";

type TvTa = Record<string, number | string | null> | null | undefined;

export default function TVConvictionPanel({
  tvTa,
  explanation,
}: {
  tvTa: TvTa;
  explanation?: string;
}) {
  if (!tvTa) {
    return (
      <div className="border rounded-lg p-4 bg-white">
        <h3 className="text-sm font-semibold text-gray-700 mb-2">
          External Confirmation (TradingView)
        </h3>
        <p className="text-sm text-gray-500 italic">TV data unavailable</p>
      </div>
    );
  }

  const rows: { key: string; label: string; raw: unknown }[] = [
    { key: "overall",     label: "Overall",    raw: tvTa["Recommend.All"] },
    { key: "ma",          label: "Moving Avgs", raw: tvTa["Recommend.MA"] },
    { key: "oscillators", label: "Oscillators", raw: tvTa["Recommend.Other"] },
  ];

  return (
    <div className="border rounded-lg p-4 bg-white">
      <h3 className="text-sm font-semibold text-gray-700 mb-3">
        External Confirmation (TradingView)
      </h3>
      <div className="space-y-2">
        {rows.map((row) => {
          const chip = classifyTvScore(row.raw);
          return (
            <div
              key={row.key}
              data-testid={`tv-row-${row.key}`}
              className={`flex items-center justify-between border rounded px-3 py-2 text-sm ${chip.className}`}
            >
              <span className="font-medium">{row.label}</span>
              <span className="flex items-center gap-2">
                <span className="tabular-nums text-xs">{chip.display}</span>
                {chip.label && (
                  <span className="text-xs font-semibold tracking-wide">
                    {chip.label.replace("_", " ")}
                  </span>
                )}
              </span>
            </div>
          );
        })}
      </div>
      {explanation && (
        <p className="text-xs text-gray-500 mt-2 italic">{explanation}</p>
      )}
    </div>
  );
}
```

test.describe("V6T-3 TVConvictionPanel", () => {

## Important Notes

- `pillar_3` is optional in the type — do not assume its presence. `stock.conviction.pillar_3?.tv_ta ?? null` is the canonical access pattern.
- The TV panel is a new sibling below `<ConvictionPillars />`, NOT a new column inside it. This keeps the existing 3-column pillar grid intact.
- Colour classes come from `classifyTvScore`'s `className` field — do not hardcode colours in the component, so V6T-4 can reuse the same helper for table mini-chips.
- Numeric scores may come in as strings if the backend ever stringifies them — `classifyTvScore` guards with `typeof raw === "number"`, treating non-numbers as null.
- No new npm dependencies. Reuse Tailwind only.
- The frontend CLAUDE.md + AGENTS.md warn that Next 16.2.3 + React 19.2.4 may differ from training data. This chunk uses only vanilla client components + hooks — no new framework APIs.
- `data-testid` attributes (`tv-row-overall`, `tv-row-ma`, `tv-row-oscillators`) are load-bearing — V6T-4 and future visual QA rely on them.
