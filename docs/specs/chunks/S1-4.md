## Objective

Convert `frontend/mockups/stock-detail.html` into a live dynamic-route Next.js
page at `/stocks/[symbol]`. Route param `symbol` replaces `${symbol}` template
variables throughout. The hero block loads first; sector param for peers query
is derived from the hero response (`_meta` or a `sector` field on the stock record).

## Domain constraints

- Read `frontend/mockups/stock-detail.html` before writing any component.
- Read `frontend/mockups/assets/stock-detail.js` before writing page logic.
- Read `node_modules/next/dist/docs/` for Next.js 16 dynamic route `[param]` API.
- `params.symbol` from Next.js App Router is accessed via `props.params` in `page.tsx`.
  Consult `node_modules/next/dist/docs/` for the exact pattern in Next.js 16.
- Sector-gated peers fetch: `useAtlasData(sector ? "/api/v1/query" : null, peerParams)`.
  Do not fetch peers until sector is known — prevents spurious empty-state flicker.
- `data-v2-deferred="true"` rec-slots and signal-playback must NOT be wired to backend.
- All monetary values (price, market cap, etc.) must use `formatCurrency` — never raw number + "₹" string concatenation.
- Chart tooltips must show exact values — no rounding beyond what format.ts provides.
