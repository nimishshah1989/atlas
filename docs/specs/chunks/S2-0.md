# S2-0 — Backend lens contract + 5-signal engine + sector aggregation

**Slice:** `docs/specs/slices/s2-drill-spine.md` (read in full before starting)
**Status:** PENDING  •  **Model:** opus  •  **Estimate:** $8 / 6h
**Depends on:** none (first chunk in S2)

## Mission

Build the shared backend contract that all S2 frontend chunks consume. No tables, no Alembic. Read-only over existing JIP + atlas tables.

## Scope

1. **`backend/services/lens_service.py`** — single `LensService` that, given `(scope, id, benchmark, period)`, returns `{rs, momentum, breadth, volume}` lens values + 5 signal types per lens. Uses the shared UQL composer (spec §17). No SQL in routes.

2. **`backend/services/signal_engine.py`** — implements the 5 signal types per slice §3.3:
   - ENTRY / EXIT / REGIME / WARN / CONFIRM
   - Thresholds loaded from `backend/config/signal_thresholds.yaml` (hot-reloadable, watched via mtime)
   - Returns `[{type, lens, fired_at, value, threshold, reason}, ...]`

3. **`backend/config/signal_thresholds.yaml`** — exact threshold values from slice §3.3 table. Comment every band.

4. **`backend/services/sector_service.py`** — sector aggregation **from stock-level data + sector mapping** (NOT sector indices). Universe filter (`NIFTY|NIFTY100|NIFTY500`) actually narrows the constituent set. Ref slice §4.4.

5. **`backend/services/conviction_service.py`** — `score(instrument_id, scope) → {score, weight_band, components}`. Score = w₁·selection + w₂·value + w₃·regime_fit. Weights tunable in `signal_thresholds.yaml`.

6. **`backend/services/regime_service.py`** — composes global + India + sector regimes. Returns deployment posture + confidence per slice §4.2.

7. **`backend/services/book_service.py`** — `holdings()`, `watchlist()`, `action_queue()`, `performance()`. Reads existing portfolio tables.

8. **`backend/config/mf_sector_map.yaml`** — curated `de_mf_classification` category → atlas sector key. Replaces v2's broken substring-match (slice §6 bug fix).

9. **5 new endpoints** (slice §6.1), each a thin wrapper:
   - `GET /api/v1/lens/{scope}/{id}`
   - `GET /api/v1/regime`
   - `GET /api/v1/leaders`
   - `GET /api/v1/sector/{key}`
   - `POST /api/v1/conviction/score`
   - `GET /api/v1/book`
