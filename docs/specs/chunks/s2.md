# Chunk S2 — standards.md rewrite (audit 53 checks, add backend + product, fold DevOps/Docs)

**Depends on:** S1
**Blocks:** S3 (v1-criteria.yaml needs the new check IDs)
**Complexity:** M (2–4h)
**Dimension:** docs (folded into architecture post-S1)

---

## Goal

Rewrite `.quality/standards.md` to match the new 7-dimension engine shipped in S1. Preserve the core 53 checks that are still useful, remove ceremony, add the backend + product dimensions with full rubrics, and ensure **every check in `.quality/checks.py` has a matching entry in `standards.md`** (and vice-versa) so the doc and code never drift.

Do NOT chase a "100+" target number. The user's instruction: "ensure those 53 cover the core of how all important checks and balances, add if anything is missed." Final count will land somewhere ~60–75 depending on how many backend checks actually stick.

## Why

`standards.md` is the human-readable contract for the scoring engine. If it drifts from `checks.py`, the rubric becomes fiction. S1 changed the engine; S2 makes the doc match. Also: the current doc is inherited from JIP Command Center and still references "four platforms" and a composite — both obsolete for ATLAS.

## Punch list

1. **Audit pass.** For each of the existing 53 checks, mark KEEP / FOLD / DROP against the S1 fold table. Reconcile any discrepancies by reading `checks.py` directly — the code is the source of truth, the doc follows.
2. **Gap pass.** Review against the Forge OS Four Laws, ATLAS System Guarantees, and the project conventions in CLAUDE.md. Add any missing check that would have caught a known past failure:
   - "No print() in production code" (Four Laws: structured logging)
   - "All routes use async def, not sync def" (project convention)
   - "All money columns are Numeric(20,4), not Float" (DB convention)
   - "Every table has created_at/updated_at tz-aware" (DB convention)
   - "Every FK column has index=True" (DB convention)
   - "No raw SQL DDL — Alembic only" (DB convention)
   - "Returns from /internal/ API are tagged data_as_of + staleness" (financial domain)
   - "No df.iterrows() or df.apply on >1K rows" (data engineering)
   - Any others surfaced by the audit.
3. **Backend dimension rubric.** Write the full rubric for each check in `.quality/dimensions/backend.py`: how it works, scoring bands, plain English, fix guidance, severity. Match the format used by existing check entries.
4. **Product dimension rubric.** Write the framing — this dim pulls from `docs/specs/v{N}-criteria.yaml`. Each criterion is one check. Score = `passed / eligible × 100`. Include a worked example (V1 criterion 7 — "≥5 decisions per pipeline run").
5. **Dimension headers.** Rewrite each dim's header section to remove weight percentages (no composite means no weights). Replace with: `Gating: true/false`, `Owned by: <who>`, `Runs: <when>`.
6. **Strip obsolete content.** Remove: the "four platforms" language, the composite formula at the bottom, the weighted sum example, the monthly API cost estimate, the "Fix with Claude" prompt templates (those belong in orchestrator/prompts.py, not the rubric). Remove the 50-69 orange / 0-49 red color bands — only binary pass/fail matters under the new gate.
7. **Consistency pass.** Every check ID in the doc must exist in `checks.py`, and every check function in `checks.py` must have an entry in the doc. Add a script: `.quality/verify_doc_matches_code.py` that grep-reconciles the two. Wire it into the `architecture` dim as a new check (`a7_standards_doc_matches_code`).
