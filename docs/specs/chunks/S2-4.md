# S2-4 — View 3: Sector Page (`/sector/[key]`)

**Slice:** `docs/specs/slices/s2-drill-spine.md` (read §4.4, §5)
**Status:** PENDING  •  **Model:** sonnet  •  **Estimate:** $4 / 4h
**Depends on:** S2-1

## Mission

Build the Sector page at `/sector/[key]`. Single screen: sector header + universe filter + 3 tabs (Stocks / MFs / ETFs) sharing the locked 11-column schema. Sector data is **derived from stock-level data** per §8 Q1 — not from a sector-index row.

## Scope

1. **Route:** `frontend/src/app/sector/[key]/page.tsx`. This REPLACES (not augments) any pre-existing `/sector/*` v1 freelance shell; retirement of the old route is S2-8's job, but do NOT leave both wired.

2. **Header card:** sector's own 4-lens summary (values come from `GET /api/v1/sector/{key}`) + signal flags + composite action + ST/MT/LT breadth mini-bars.

3. **Universe filter:** `NIFTY | NIFTY100 | NIFTY500` — toggle changes the `universe` query param, backend re-aggregates constituents (§4.4). Default NIFTY500.

4. **Three tabs:**
   - **Stocks** (default) — `<InstrumentTable mode="stock">` sorted by composite action severity.
   - **Mutual Funds** — `<InstrumentTable mode="mf">` using the curated `mf_sector_map.yaml` from S2-0 (fixes v2 substring-match bug, §8 Q7).
   - **ETFs** — `<InstrumentTable mode="etf">`. If zero → `<EmptyState>` ("No ETFs mapped to this sector").

5. **Row click** → respective instrument detail page (existing `/stocks/[symbol]`, `/etf/[ticker]`, `/funds/[id]`, upgraded by S2-5).

## Hard stops

- Fetching `/api/v1/sector/{key}` for an unknown sector key → backend should 404, frontend must render a friendly "unknown sector" state, NOT crash
- Reading from a `de_sector_indices_*` table for bubble values → STOP, violates §8 Q1
- Inventing an ETF when sector has none → STOP, empty state is correct
