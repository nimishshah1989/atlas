**Status:** PENDING  •  **Model:** sonnet  •  **Estimate:** $5 / 5h
**Depends on:** S2-1, S2-3

## Mission

Fill in the real content behind the `<ConvictionDrawer>` skeleton from S2-1. Wires to `POST /api/v1/conviction/score` (already built in S2-0). Triggered from "Size it" on Leaders Board (S2-3) and, later, Book (S2-7). Non-blocking — closing never loses context.

## Scope

1. **Drawer body per instrument:**
   - **Fundamentals stack** (per asset class — stock: P/E zone, earnings mom, FCF, leverage, RoCE; ETF: expense, AUM, concentration, premium/discount; MF: NAV, AUM, expense, fund-mgr tenure, top holdings).
   - **Conviction score 0-100** with visible decomposition: `w₁·selection + w₂·value + w₃·regime_fit` — show each component value + weight.
   - **Suggested weight band**: 1% / 3% / 5% bucket, with rule shown ("RICH zone caps at 1% even if RS top decile").
   - **Portfolio impact preview**: "Adding at X% raises sector exposure to Y%, factor tilt to Z" — reads current holdings from `/api/v1/book.holdings`.

2. **POST wiring:** `POST /api/v1/conviction/score` body `{instrument_id, scope}` → render response. Re-fetch on prop change (SWR with mutate).

3. **Weights tunable:** display current `w₁/w₂/w₃` from `signal_thresholds.yaml` read-only. No UI to edit.

4. **"Add to watchlist"** button inside drawer — POSTs to existing watchlist endpoint (or S2-0's book service if that's where it lives). If endpoint missing → disable button + tooltip, don't fail.

5. **Replace skeleton** from S2-1 (`ConvictionDrawer` now has real content). Keep the open/close contract unchanged so Leaders Board callers don't break.
