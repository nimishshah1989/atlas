## Domain constraints

- Read `frontend/mockups/mf-rank.html` before writing any component.
- Read `node_modules/next/dist/docs/` for Next.js 16 API before writing any page.
- `data-block="rank-table"` must carry `data-endpoint="/api/v1/query/template"` via
  equivalent React data attribute if E2E tests check for it.
- Sparklines MUST use batched data from the rank response — do NOT issue N individual
  `/api/v1/mf/${id}/nav-history` calls. The mf_rank_composite template includes sparkline
  data in its response.
- Filter state lives in `page.tsx` — not in URL search params (for this chunk).
  Future enhancement may add URL sync.
- CSV export: use `URL.createObjectURL(new Blob([csv], { type: "text/csv" }))` + programmatic anchor click.
  No third-party CSV library.
- Return values: `formatPercent` with sign. Positive = text-emerald-600, negative = text-red-600.
- Composite score: `formatDecimal(val, 2)` — 2 decimal places, right-aligned.
