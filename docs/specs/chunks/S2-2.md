## Mission

Build the Regime Cockpit at `/` — the PM's "timing" view. Three stacked bands, no tabs. Single deployment posture banner at top. Sector heatmap drives drill into `/sector/[key]`.

## Scope

1. **Route:** `frontend/src/app/page.tsx` replaces the current root (was a redirect to `/pulse` per S1-7). Ref memory `project_s1_frontend_sprint.md` — update redirect chain.

2. **Posture banner:** single sentence across the top per slice §4.2 ("Deployment posture today: **SELECTIVE** — Global Risk-On, India CAUTIOUS, breadth divergent ST↔LT") + confidence %. Reads from `GET /api/v1/regime`.

3. **Band 1 — Global:** 4 region cards (US / EU / EM / India) with regime badge + composite score. Source: `regime.global` from S2-0.

4. **Band 2 — India:** `<RegimeGauge>` 0-100 composite + 5 mini-gauges (short-term / broad-trend / a/d / momentum / extremes) + explicit ST(10d) / MT(50d) / LT(200d) breadth bars.

5. **Band 3 — Sector heatmap:** rows = sectors, columns = ST / MT / LT, cell color = breadth state (green ≥60, amber 40-60, red <40). Click cell → `router.push('/sector/[key]')`.

6. **Methodology drawer:** "How regime is computed" slides in from right. Read-only copy of composite formula + threshold bands from `signal_thresholds.yaml`.
